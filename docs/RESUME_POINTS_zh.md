# 简历要点

选择最适合你简历布局的要点。

## 项目: 高性能 C++ 键值缓存
*技术栈: C++20, Linux, Epoll, TCP/IP, Docker, Google Benchmark*

### 选项 1: 侧重性能
*   设计并实现了一个基于 **C++20** 的高吞吐量键值存储，利用 **Epoll** 进行非阻塞 I/O 以处理并发连接。
*   通过从全局互斥锁迁移到 **分片 LRU** 架构优化了并发性，减少了锁竞争，在多线程基准测试中实现了 **8.4 倍的性能提升**。
*   设计了自定义 **二进制协议** (Header+Body)，与基于文本的协议相比，最小化了网络开销和解析延迟。
*   实现了 **AOF (Append-Only File)** 持久化机制，以确保数据持久性和崩溃恢复。

### 选项 2: 侧重系统工程
*   使用 `std::list` 和 `std::unordered_map` 从头构建了线程安全的 **LRU 缓存**，操作时间复杂度为 O(1)。
*   使用 Reactor 模式和 **Epoll** 边缘触发开发了 **TCP 服务器**，以在 Linux 上进行高效的事件处理。
*   集成了 **Google Benchmark** 和 **GTest** 进行严格的性能分析和单元测试。
*   使用 **Docker** (多阶段构建) 对应用程序进行了容器化，并设置了 **GitHub Actions** 用于自动化 CI/CD 管道。

### 选项 3: 简洁 / 一句话
*   **C++ KV Cache**: 构建了一个具有自定义二进制协议和 AOF 持久化的高性能、分片键值存储；通过锁分片实现了 8.4 倍的并发加速。

## 展示的关键技能
*   **C++**: RAII, 智能指针, 模板, 移动语义, STL。
*   **并发**: 互斥锁, 锁粒度, 线程池, 竞争条件处理。
*   **系统**: Socket 编程, 文件 I/O, 内存管理。
*   **工具**: CMake/Xmake, Docker, Git, GDB, Valgrind。
